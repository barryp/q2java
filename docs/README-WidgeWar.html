WidgetWars

(a working title)

Server QuickStart

WidgetWars is a team game that uses CTF maps and resources. To make those resources accessible to your server, you'll have to copy or move the CTF .pak file from your quake2\ctf directory to your quake2\baseq2 directory (changing the number of the pak file if there's already a .pak file in your baseq2 directory with the same number).

Start the game with a command line such as:

q2java +map q2ctf1 +set q2java_startup widgetwar.startup.sample

Add +set dedicated 1 for a dedicated server. Edit the widgetwar.startup.sample file to customize the game. If you call your startup file just q2java.startup, you can get by with the shorter command-line:

q2java +map q2ctf1

Client QuickStart

Game Resources

WidgetWars is a team game that uses CTF maps and resources. To make those resources accessible in your client, you'll have to copy or move the CTF .pak file from your quake2\ctf directory to your quake2\baseq2 directory (changing the number of the pak file if there's already a .pak file in your baseq2 directory with the same number).

There are also some screen icons you need to download from the server the first time you play the game. You can do this by starting the game with this command:

quake2 +set allow_downloads 1 +connect www.xxx.yyy.zzz

Where www.xxx.yyy.zzz is the IP address of the WidgetWar server. Once the icons are downloaded to your disk, you don't need to do the allow_downloads bit anymore, and can start q2 without that extra +set allow_downloads 1 parameter.

Commands

The commands unique to this game are:

  • widget
  • body
  • control
  • payload
  • deploy
  • describe

The first five will be frequently used, so you'll probably want to bind them to keys. I've been using this binding:

bind z widget
bind x body
bind c control
bind v payload
bind b deploy

You can type those lines in your console, or place them in a file named quake2\q2java\autoexec.cfg if you don't want to type them every time you connect.

I'll get into what they do later in this document.

Playing the Game

The Objective

The main objective in this game, is to steal enemy technology, and prevent the enemy team from stealing your technology. Technology is represented by CDs (compact-disks) that can be found where flags normally appear in CTF - or by DataCards occasionally dropped by dying players.

If you bring a CD or DataCard back to your base (where the CTF flags would appear), you'll capture some fraction of an enemy technology. The amount captured on a CD depends on how long it took you to make the capture. The amount captured on a Datacard is fixed, but if a player dies while carrying an enemy datacard, that card is lost (unlike a CD which can be picked up by a teammate and carried on to the base)

Once a team has captured 100% of a technology, the team can start using that technology.

The game also keeps score based on percentage of technologies captured. If a team doesn't have any more technology to steal, then any further captures are basically just for the points - which show up on the lower right of the HUD - one point for each percent of technology captured.

Technologies

What's a technology, and why would you want to capture one? Unlike a CTF flag which is just a symbolic token worth points on a scoreboard, a technology is actually useful. A technology is the knowledge of how to build a widget component. The more technologies a team has, the greater the variety of items team members can build.

When a level first starts, both teams are given some common technologies, and a few unique ones. As the level progresses, every 5 minutes or so teams are given new technologies - so there are new things to steal, and the game just gets more and more intense.

Widgets

A widget is an item you build and deploy into the world, made up of a combination of various components. Each component is an instance of a particular technology. There are three types of technologies/components:

Body: responsible for the appearance and movement of the widget in the world. For example, a simple box that sits on the ground, or a guided missile.

Control: (optional) gives some direction to the body and/or payload. For example, one control unit might automatically target enemy players, another might target enemy widgets.

Payload: (optional) hopefully does something useful - like kill enemy players. For example, a cluster bomb that might throw out a shower of grenades, or a comm jammer that screws up enemy team communications.

To create a widget, you select a body using the body command (or by tapping whatever key you've bound that command to), select a control with the control command, and a payload with the payload command. The lower-left corner of the HUD (Heads Up Display) shows you current selection of components. Keep tapping the keys until you come up with the combination you want, and then hit the deploy key (which I bound to "b" in my example bindings at the top of this doc).

On executing the deploy command, your widget should appear in front of you. What it does after that depends on the particular combination of components you selected (a box will probably just sit there, while a missile will probably launch immediately). On your HUD, above the component display, you'll see a row of box-indicators, one for each widget slot. The "current" widget is indicated by a yellow highlight, and an green-light indicates this widget is active.

If you want to create another widget, you'll have to select another widget-slot, use the widget command to rotate through the various slots, until you find one that's empty (has no green light).

If you want to destroy a widget to free up a slot, use the widget command to rotate through the slots until you come to the one you want to get rid of, and then tap the deploy key twice (the first time you tap it you'll hear a warning noise).

The last command you might want to know about is describe - which prints on the console a short description of what the currently selected technologies do.

Energy

Energy is the main resource in this game. Instead of individual types of ammo, all the weapons use energy cells. Energy is also consumed when you build widgets and when you jump (bunny-hoppers beware).

Where does energy come from? Your battle suit naturally generates energy, more as you're standing still, less as you're running. You can use the drop cells command to give energy to other players, and dying players will sometimes drop energy cells.

There will also be energy-related technologies in the game, such as the "energy siphon" payload which steals energy from enemy players.

Health

Health is similar to Energy in that your suit will automatically heal you, but only if you're standing still. There will also be health-related technologies in the game such as the "AutoDoc" payload

Lastly

You won't find any spawned items such as weapons, ammo, armor or powerups. You start off right away with all the weapons you'll ever have - which are the normal deathmatch weapons up to the hyperblaster (railgun and BFG10K are left out). The only things that are spawned are the two CDs in the team flagrooms.

Known Bugs

This game is not complete - there are quite a few things that need work.

  • Widgets may be created sticking out of walls
  • The balance of various game elements is very rough - basically there need to be adjustments to the energy cost of each item, the speed and potency of various weapons and widget components, and various other game factors relating to timing and scorekeeping.

Examples

There will be a huge number of ways technologies can be combined to create widgets, but I'll list a few example combinations here to give you some idea of what's possible.

Current Technologies

These are some of the technologies currently implemented in the game:

  • Bodies
    • Box - just sits on the ground, pretty simple
    • Body Harness - lets you attach other components to your own body. Has the special feature that any control components show their output on the upper-right of your HUD with direction and range indicators.
    • Guided Missile - flies towards whatever spot is specified by a control component (flies in a straight line if no control component is present, or the control hasn't specified a target).
    • Standby Missile - like a Guided Missile, but doesn't launch until a target is detected.
  • Controls
    • Enemy Tracker - tracks the nearest enemy player
    • Enemy Carrier Tracker - tracks the nearest enemy player carrying stolen Tech.
    • Enemy Widget Tracker - tracks the nearest enemy widget
    • Laser Guidance - tracks whatever spot the player's crosshair is resting on, when the fire button is pushed. As a convenience, the Blaster weapon has been replaced with a null-weapon - so that when you select the blaster, you actually have no weapon at all, and can safely press the fire button without actually firing anything and giving your position away
  • Payloads
    • AutoDoc - automatically heals nearby team members
    • Cluster Bomb - when triggered it throws out a shower of grenades
    • Comm Jammer - disrupts enemy team communications originating near the widget
    • Echelon Communications Interceptor - eavesdrop on enemy communications
    • Energy Siphon - steals energy from enemy players
    • Explosive - simple thing that goes "boom!"
    • Forcefield - take a wild guess..

There will eventually be many more technologies, but that's what I have to start with.

Example Combinations

These are just a few example combinations - there are quite a few more that are possible. Not all combinations of components will be useful though - it'll be up to each player to figure out for themselves what works, and what is a waste of energy.

Box + AutoDoc = First aid station. The box will simple sit on the ground, and any team member that stands near it will be healed.

Body Harness + AutoDoc = Medic. Team members standing near a player wearing an AutoDoc will be healed.

Box + Cluster Bomb = Remotely triggered bomb. Create a widget like this, then step back and hit the deploy key twice - kaboom!

Box + Enemy Tracker + Cluster Bomb = Automatically triggered bomb. Set one of these up, and it will go off when an enemy player comes near.

Guided Missile + Enemy Tracker + Cluster Bomb = A missile that flies off towards an enemy player, and rains a shower of grenades on his head when it comes close enough.

Guilded Missile + Laser Guidance + Cluster Bomb = A missile that flies towards whatever spot you point your crosshair at while pressing the fire button. If your target is moving, just keep pointing the crosshair at him, and the missile will steer towards him.

Body Harness + Enemy Tracker = Enemy Detector, will show on your HUD the direction and range of a nearby enemy player.

Box + Energy Siphon = More juice for you, if you can sneak this into an area where enemy players tend to congregate.
